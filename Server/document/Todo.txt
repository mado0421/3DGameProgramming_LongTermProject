현재 회전정보는 사용 안하고있음

클라에서 유저가 들어오고 나갈 수 있다
	=> 현재 위치값이 없음 전부 클라의 시작위치 값을 사용하고
	다른 위치도 고정이라 모두 화면은 같은데 좌표계상은 어긋나게 움직임
	=> Server에도 클라의 위치정보를 써야한다 회전이나 나중 OOBB 충돌등을 생각하면
	클라에서 사용중인 DXMatrix 사용하는 것이 충돌처리등 기존의 로직을 가져오기 쉬울것

벽 충돌체크를 할 수 없다 클라에서 충돌체크를 해도
서버에서는 체크를 안하고 움직임에 이동값을 주기 때문에
서버에서도 충돌체크를 할 수 밖에 없을듯 확인해봐야함

다른 사물들의 관리 문제
현재는 Scene에 있는 유저를 새로 추가하거나 나갈때
하위 클래스 ClientWsaModule(CWM)에서 통신을 처리하고
해당 오브젝트들의 포인터와 Scene의 포인터도 Cwm에서 갖고있다
그 이외에 타겟보드(타켓) 및 총알(?)등 오브젝트의 통신은 이뤄지지않음


인풋 문제?
현재 편하게 애니메이션 관리를 위해 클라와 서버의 처리는

 클라				
키입력 -> 키버퍼 -> 키버퍼읽고 무브 packet send 

  서버				클라
->처리 -> 무브패킷 클라에 전송 ->무브를 담당하는 KeyBuffer를 셋팅

의 방식으로 이뤄지고 있다. 실제의 포지션 값을 움직이기보다
기존에 클라에서 받는 input을 서버에서 입력한다는 느낌으로 구현되어있음
구현의 편리성으로 이런 방식을 취했지만 문제가 있을수도 있다.
=> 서버에서도 포지션을 관리하여 확실하게 하면 해결되지 않을까 생각됨
